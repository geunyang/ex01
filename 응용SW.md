# 운영체제 기초 활용하기
## 운영체제 특징 파악
### 운영체제
* 사용자가 컴퓨터 하드웨어를 쉽게 사용할 수 있도록 인터페이스를 제공하는 소프트웨어
### 윈도우즈
* 1995 년도 출시
* 사용자가 컨트롤하는 마우스의 아이콘을 이용하여 소프트웨어 실행(GUI)
* 마이크로소프트사만이 수정 및 배포 가능
* 고객 지원이 체계적이나 문제 수정에 시간이 걸려 유닉스에 비해 보안 취약
### 유닉스
* 1960년대 탄생
* 멀티태스크 기능에 초첨
* 대부분 유료
* 서버와 슈퍼컴퓨터 같은 대형 서비스 시장에서 높은 점유율
* 커맨드 기반
* Bourne Shell 사용
### 리눅스
* 유닉스의 호환 커널로 1991 리누스 토발즈에 의해 개발
* 자유 소프트웨어 정책 하에 대부분 자유롭게 사용 가능하고 재배포 가능
* GUI 제공
* Bash Shell 사용
* 모바일, 태블릿 등 범용성 높음
* 누구나 접근 가능해 문제 발생시 빠른 업데이트 가능하나 윈도우즈와 같은 체계적 지원은 부족하여 전문가 위주로 사용
## 운영체제 기본 명령어 활용
### CLI
* 사용자가 직접 명령어를 입력하여 컴퓨터에게 명령을 내리는 방식
### CLI 기본 명령어 활용
* 윈도우즈 : 파일검색->CMD(또는 보조프로그램->Command창 호출)->Help 입력해 명령어 검색
* 리눅스 : Shell 창 호출 ->'-help','-h','#man' 입력해 명령어 검색
### GUI
* 마우스로 화면을 클릭하여 컴퓨터를 제어하는 방식
### GUI 기본 명령어 활용
* 윈도우즈 : 제어판->시스템및 보안->문제찾기및해결->도움말및지원->기본명령어 도움말 찾기
* 리눅스 : help center -> 기능 확인 및 실행
## 운영체제 핵심 기능 파악
### 메모리 관리 수행
* CPU 와 지속적으로 데이터를 송수신 하는 상황에 입출력의 순서나 메모리 공간 확보등을 진행한다
### 프로세스 관리 수행
* 프로세스는 실행중인 프로그램을 의미한다
* 관리 기법 일시중지 및 재실행, 동기화, 통신, 교착상태처리, 프로세스 생성&삭제
### 클라우드 환경 세팅 이해 및 수행
* 인터넷 기반에서 구동되는 컴퓨터 기술을 의미
* 데이터를 손쉽게 저장하고 불러올수 있다
* Iaas : 구글, 아마존 등에서 제공하는 인프라를 임대하여 이용
* Paas : 운영체제가 이미 구성되어 있고 데이터와 애플리케이션만 직접 관리할수 있음
* Saas : 인프라, 운영체제, 소프트웨어까지 갖추어진 서비스
# 네트워크 기초 활용하기
## 네트워크 계층 구조 파악
### OSI7 레이어 각 층의 역할 및 주요 네트워크 장비
* 7. 응용계층 : 사용자 친화 환경 제공. 이메일, 웹 등(pc)
* 6. 표현계층 : 코드 문자 등을 번역하여 일관되게 전송하고 압축, 해제, 보안 기능 담당(pc)
* 5. 세션계층 : 송신, 수신 간의 논리적 연결(pc)
* 4. 전송계층 : 송신, 수신, 프로세스 간의 연결(TCP/IP UDP)
* 3. 네트워크계층 : 다수의 중개 시스템 중 올바른 경로를 선택하도록 지원(라우터)
* 2. 데이터링크계층 : 오류와 흐름을 제거하여 신뢰성 있는 데이터를 전송(브릿지, 스위치)
* 1. 물리계층 : 실제 장비들을 연결하기 위한 연결 장치(허브, 리피터)
### 인터넷망 구성의 이해
* WAN(Wide Area Network) : 광대역 네트워크. 국가, 대륙과 같이 광범위한 지역 연결. 거리 제약 없으나 속도 느리고 에러율 높음. 회선교환방식, 패킷교환방식 
* LAN : 근거리 네트워크. 한 건물 또는 작은 지역을 커버
## 네트워크 프로토콜 파악
### 네트워크 프로토콜의 특징
* 단편화 : 전송이 가능한 작은 블록으로 나누어지는것
* 재조립 : 단편화되어 온 조각들을 우너래 데이터로 복원하는 것
* 캡슐화 : 상위 계측의 데이터에 정보를 추가하여 하위 계층으로 보내는 것
* 연결 제어 : 데이터의 전송량이나 속도를 제어하는 것
* 오류 제어 : 전송 중 잃어버리는 데이터나 오류가 발생한 데이터를 검증하는 것
* 동기화 : 송신과 수신 측의 시점을 맞추는 것
* 다중화 : 하나의 통신 회선에 여러 기기들이 접속할 수 있는 기술
* 주소 지정 : 송수신지에 주소를 부여하여 정확산 전송을 보장하는 것
### 전송 계층의 TCP/UDP 차이
* TCP : 연결형 서비스, 가상회선, 전송 순서 보장, 수신여부 확인, 1:1 통신, 신뢰성 높음, 느린 속도
* UDP : 비연결형 서비스, 데이터그램 방식, 전송 순서 바뀔 수 있음, 수신여부 확인하지 않음, 1:1/1:N/N:N 통신, 신뢰성 낮음, 빠른 속도
## 네트워크 핵심 알고리즘 파악
### 스위칭
* 네트워크에서 노드와 노드 간 연결을 중개해주는것
* 직접 연결 구축 불가
* 전화 교환원과 같은 역할
### 라우팅 알고리즘
* 데이터가 목적까지 닿는 최적 경로를 산출하기 위한 법칙
* 거리 벡터 알고리즘 : 라우터 간의 최단 경로 스패닝 트리를 찾고 이용 불가시 다른 경로를 찾는다. 계산면에서 단순
* 링크 상태 알고리즘 : 라우더 간의 모든 경로를 파악한 뒤 대체 경로를 사전에 마련. 일관성 있는 네트워크 파악이 가능하나 계산이 복잡하고 트래픽이 광범위해진다
### IP 프로토콜
* 전세계 컴퓨터에 부여되는 유일한 식별자
* IPv4 : 32비트, 8비트 4부분 10진수, 43억개, 클래스 단위의 비순차적 할당, 품질제어불가, 보안프로토콜별도설치, 플러그앤드플레이불가, 모바일 어려움, 웹캐스팅 어려움
* IPv6 : 126비트, 16비트 8부분 16진수, 약 43억x43억x43억x43억개, 네트워크 규모 및 단말기 수에 따른 순차 할당, 등급별서비스별 패킷 구분 가능해 품질보장 용이, 확장 기능으로 보안 제공, 플러그앤드플레이 지원, 모바일ip 용이, 웹캐스팅 용이
# 기본 개발환경 구축하기
## 운용체제 설치 및 운용
### 윈도즈와 리눅스 운영체제의 특징
* 윈도우즈의 특징 : 개인용, 소규모 기업용, 복구 기능을 필요로 하는 소규모 기업용 운영체제
* 리눅스의 특징 :  유닉스와 완벽 호환, 공개 운영체제, pc용 os보다 안정적, 다중 작업 가능, 한대의 시스템을 많은 작업자가 동시 사용가능, 적은 양의 메모리를 필요로함
### 다양한 버전의 리눅스
* Devian GNU : 개발자 패키지 및 매뉴얼 활성화로 개발자 최적화
* ubuntu : 가장 널리 사용됨, 응용 소프트웨어 공급
* openSUSE : 안정화 버전과 테스트 버전 동시 배포, 태스크 자동화 지원
* Fedora : 스마트 설정 및 업데이트, 안정화, 다양한 하드웨어 지원
* sentOS : 프로그래밍에 최적화, RHEL 소스로 컴파일
* slackware : 다양한 소프트 웨어, GUI설치, 복구툴

## 응용 시스템 개발 인프라
### 개발환경 인프라 방식
* On-Premise : 외부 인터넷망이 차단된 상태에서 인트라넷 망만을 사용하여 개발 환경 구축. 정보의 외부 유출에 민감할 경우 사용
* 클라우드 : 아마존 등 클라우드 공급 업체의 서비스를 임대하여 개발 환경 구축. 개발 환경 투자비용이 적고 구축시간이 빠름
### 클라우드 기반 개발환경
* 컴퓨팅 환경 : 하드웨어 장비 세팅
* 스토리지 : 대규모 데이터 저장, 관리, 전송, 이중화 장치 세팅
* 데이터베이스 : 실데이터 저장, 관리할 데이터베이스 세팅
* 네트워킹 전송 : 구성된 서비스 및 콘텐츠를 전달하기 위한 환경 세팅
* 개발자 도구 : 실제 개발을 위한 환경 구축(개발,테스트,배포,형상관리 등)
* 보안 환경 구축 : 침입으로 부터 시스템과 데이터를 보호(암호화,자격증명,인증서 등)
* 응용 기술 세팅 : 증강현실, 머신러닝, 사물인터넷 등
* 생산성 향상 : 시스템볼륨, 스트리밍, 비즈니스, 운영상태 분석 등
